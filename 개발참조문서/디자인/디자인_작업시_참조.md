🎨 Web Design Guidelines for Vibe Coding
이 문서는 AI 기반 코딩(Vibe Coding) 시 디자인 일관성을 유지하고 전문가 수준의 UI/UX 품질을 확보하기 위한 핵심 규칙을 정의합니다. 모든 코드 생성 및 UI 설계 시 이 가이드라인을 최우선으로 참고하십시오.
1. Color System (색상 규칙)
디자인의 조화로움을 위해 다음의 엄격한 색상 제한 규칙을 따릅니다.
• Color Selection: 직접 색상을 고르지 말고, 반드시 Coolors.co 등에서 추출한 검증된 Hex 코드를 사용합니다.
• The Power of Two: 유채색은 딱 두 가지만 사용합니다.
    ◦ Main Color (Primary): CTA 버튼, 핵심 강조 요소에만 사용.
    ◦ Sub Color (Secondary): 배경이나 보조적인 장식 요소에만 사용.
• Neutral Background: 위 두 색상을 제외한 나머지는 전부 **무채색(흰색, 검정색, 회색 계열)**으로만 채웁니다.
• No Modification: 제공된 Hex 코드를 임의로 진하게 하거나 수정하지 말고 그대로 적용합니다.
2. Component Library (UI 라이브러리)
Next.js 프로젝트 진행 시 AI와의 협업 효율성을 극대화하기 위해 다음 도구를 사용합니다.
• shadcn/ui 추천: MUI 대신 shadcn/ui를 사용합니다.
    ◦ 이유: 소스 코드가 프로젝트 내부에 직접 위치하므로 AI가 코드를 직접 읽고 수정하기 가장 적합합니다.
• Tailwind Config: 프로젝트 시작 시 tailwind.config.js의 theme.extend에 지정된 메인/서브 컬러를 primary, secondary로 미리 설정하십시오.
3. UI/UX Execution Process (실행 프로세스)
화면 레이아웃과 구성을 요청할 때는 다음의 절차를 준수합니다.
• English-Based Layout: 레이아웃 묘사나 레퍼런스 사이트 설명은 반드시 구체적인 영문 프롬프트를 사용합니다. (한국어보다 AI의 이해도가 훨씬 높음)
• Prompt Extraction: 레이아웃 묘사가 어려울 경우, 먼저 클로드에게 구현하려는 서비스의 특징을 설명하고 '상세 영문 레이아웃 프롬프트'를 추출해달라고 요청하십시오.
• Stitch 활용: UI 구조 생성 시 'Stitch'와 같은 도구를 활용하되, 이때 준비된 정확한 Hex 코드를 함께 전달하여 컬러 일관성을 유지합니다.
4. Efficiency & Consistency (효율성 및 일관성)
• Anti-MCP for Design: 피그마 MCP 등을 통한 직접적인 디자인 조작은 지양합니다. 토큰 소모가 크고 맥락이 끊겨 디자인이 무너질 수 있습니다.
• Skills Over MCP: 이 가이드라인 문서를 **스킬(Skills)**로 참조하여, 매번 API를 호출하지 않고 텍스트 문서를 기반으로 일관된 규칙을 적용하게 합니다.
• Early Setting: 기능 구현 후 디자인을 입히는 것이 아니라, 처음부터 컬러와 shadcn/ui 세팅을 완료한 후 코딩을 시작합니다.

💡 클로드 코드에게 스킬을 적용하는 법 (명령 예시)
클로드 코드에게 작업을 시킬 때 다음과 같이 입력하여 위 가이드라인을 활성화하세요.
"지금부터 Next.js와 shadcn/ui를 사용해 프로젝트를 진행할 거야. web-design-guidelines.md에 적힌 디자인 스킬을 엄격히 준수해줘. 메인 컬러는 #007AFF(파란색), 서브 컬러는 #E5F1FF(연한 파란색)야. 이 규칙에 맞춰서 첫 번째 랜딩 페이지 레이아웃을 잡아줘."
이렇게 스킬을 문서화해두면 클로드가 대화 도중 디자인 규칙을 잊어버리는 것을 방지하고, 토큰 소모를 줄이면서도 일관된 품질의 결과물을 만들어낼 수 있습니다.
